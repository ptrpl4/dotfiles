---
name: explain
description: Explain the current file, function, or code selection concisely. Use when the user wants to understand unfamiliar code without making changes.
---

Explain the code clearly and concisely. Tailor depth to complexity — simple code gets one sentence, complex code gets a structured breakdown.

1. Read the file or selection in context
2. State what it does in one sentence
3. If non-trivial, cover: inputs/outputs, key logic, side effects, and anything surprising or non-obvious
4. Note any dependencies or callers worth knowing about
5. Do not suggest changes unless explicitly asked
